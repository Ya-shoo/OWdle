# Share cards & the OG renderer (`/r/` links)

> Deep dive referenced from `AGENTS.md`. Read when touching `functions/og/r/[code].tsx`, `functions/r/[code].ts`, `lib/shareUrl.ts`, `lib/shareLinks.ts`, or the share UI. The renderer's hardening each guards a shipped incident — **do not simplify it away.** The general cross-site share-card model + "port a fix between sites" procedure live in the `dailydles` skill.

## The link-first system
Round results and the daily summary share via `/r/[code]` links that unfurl into server-rendered spray-style cards.

- `lib/shareUrl.ts` — compact code encode/decode. Bundled into Pages Functions: keep it free of app imports and of anything touching `process`. Daily code `YYMMDD-<5 mode chars>-<hints><skips>`; round code `YYMMDD<letter><result>`. `ROUND_MODE_CHAR`: classic `c`, quote `q`, splash `p`, sound `s`, ability `a`, map `m`. Mode/slot order follows `BUILT_MODE_SLUGS`.
- `lib/shareLinks.ts` — builds `{url, ogImageUrl}` via `roundShareLinks` / `dailyShareLinks` → `SITE_URL/r/<code>/` + `/og/r/<code>`; `ogPreviewSrc` / `ogRetrySrc` handle retries.
- `functions/r/[code].ts` — unfurl HTML shell (meta-refresh to `/{mode}/?c=`).
- `functions/og/r/[code].tsx` — the `workers-og` card renderer.

## OG renderer hardening (ported from Deadlockle, 2026-06-06)
Keep the two repos in lockstep.

- **Renders persist to R2** (`OG_CACHE` → shared `dailydles` bucket, keys `og-cache/owdle/<REV>/<code>.png`): free-plan Workers CPU limits kill ~half of COLD wasm renders (lazy resvg/yoga init inside the request), so each code renders once and every later request serves storage. **Bump `RENDER_REV` in `functions/og/r/[code].tsx` whenever the card design changes** — stored renders are immortal. Local dev bypasses storage (live render always).
- **Fonts are self-hosted subsets** (`public/og-fonts/`, regenerated by `scripts/fetch-og-fonts.mjs`) — the render path never leaves the zone. If a card ever renders a NEW glyph, add it to that script's SUBSET and re-run, or it draws as tofu. (✓/✕ are inline SVG, not text.)
- The sharer's prefetch (`ShareButton`) and the modal preview (`ShareModal`) both RETRY failed loads — the sharer's device shoulders the one cold render each code needs. **Retries fetch DISTINCT URLs** (`ogRetrySrc` adds `?r=N`): WebKit replays a same-URL image failure from its in-session memory cache without re-requesting (no-store notwithstanding), which silently made every retry a no-op on iOS — daily-summary codes (per-player unique, never prewarmed) showed "Preview unavailable" and went un-warmed for unfurlers. The query param is invisible server-side (R2 keys on the path code alone).
- `.github/workflows/prewarm-og.yml` renders all ~98 possible round codes daily after the 2:15am reset (dual UTC crons for DST). Daily-summary codes stay on-demand.
- The announce modal embeds a PRE-BAKED card (`public/announce-example.png`), not a live render. Regenerate after design changes:
  `curl -s https://playowdle.com/og/r/260605-32432-00 -o public/announce-example.png`
