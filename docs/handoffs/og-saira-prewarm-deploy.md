## The deploy-full-tree trap (read before touching bucket B)
`npm run deploy:live` = `sync-to-r2 → build:deploy (next build of the WHOLE dirty tree → out/) → wrangler pages deploy → git push`.
It ships everything in the working tree. With 75 dirty files of unready WIP present, a blind
`deploy:live` would push the Workshop redesign + Archive Sound to prod. See memory
`feedback_deploy_full_tree.md` (and `feedback_deploy_only_on_explicit_deploy.md`).

Because Cloudflare Pages deploys are atomic full-site uploads, bucket B **cannot** be shipped
without a full `out/` build. To isolate it you must build `out/` from the **current live
source state**, not the dirty tree — and only Yash knows which of the 75 uncommitted files
are already-live vs unready. **So do bucket B WITH Yash:** either (a) fold B into the next
intended full Workshop deploy, or (b) build/deploy from a scoped worktree that has current-live
+ these 3 files only. Do not guess the baseline.

Bucket A has none of this coupling — that's why it ships independently and first.

## Root cause (context — full write-up in memory `og_prewarm_guard_selfskip.md`)
- **The Cloudflare cache purge did NOT delete any OG cards.** Renders live in R2 (bucket
  `dailydles`, `og-cache/owdle/<rev>/<code>.png`), a different system from the CDN edge cache
  that "Purge Cache" clears. OG responses are `cf-cache-status: DYNAMIC` (never edge-cached),
  so a purge has nothing of theirs to touch. Verified R2 intact by pulling a stored render.
- **Real cause of flaky unfurls:** the `prewarm-og.yml` warm job silently self-skipped for
  weeks. Its DST guard required firing at exactly Pacific hour 02; GitHub delays scheduled
  crons 1-3h, so every run arrived at 03-05 PT and exited early — green in 5-9s, zero cards
  warmed. With R2 not topped up, uncached codes fell back to the free-plan cold-render 503
  lottery. All three sites had the identical guard. Fixed to a post-reset **window** (02-08 PT).
